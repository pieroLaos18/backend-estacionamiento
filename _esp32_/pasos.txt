PASOS PARA CONFIGURAR Y CORRER EL ESP32

Entras a la carpeta _esp32_, abres la carpeta now_esp32 y abres el archivo now_esp32.ino

1. En la LÍNEA 30 cambia el número de plazas según tu hardware:
   - Para 1 plaza solamente:  #define HARDWARE_COMPLETO 0
   - Para 3 plazas completas: #define HARDWARE_COMPLETO 1

2. En las LÍNEAS 66-69, cambia a TUS credenciales de HiveMQ Cloud:

   const char* mqttServer = "TU_SERVIDOR.s1.eu.hivemq.cloud";
   const int mqttPort = 8883;
   const char* mqttUser = "TU_USUARIO";
   const char* mqttPassword = "TU_CONTRASEÑA";

   IMPORTANTE: El ESP32 usa el puerto 8883 (MQTT/TLS).
   El frontend web usa el puerto 8884 (WebSocket Secure).

3. Verifica que los pines de los sensores coincidan con tu conexión física:
   - Plaza 1: TRIG_PIN_1 (GPIO 4), ECHO_PIN_1 (GPIO 5)
   - Plaza 2: TRIG_PIN_2 (GPIO 16), ECHO_PIN_2 (GPIO 17)  [solo si HARDWARE_COMPLETO = 1]
   - Plaza 3: TRIG_PIN_3 (GPIO 18), ECHO_PIN_3 (GPIO 19)  [solo si HARDWARE_COMPLETO = 1]
   - IR Entrada: GPIO 15
   - IR Salida: GPIO 14  [solo si HARDWARE_COMPLETO = 1]
   - Servo Entrada: GPIO 13
   - Servo Salida: GPIO 12  [solo si HARDWARE_COMPLETO = 1]

4. Instala las librerías necesarias en Arduino IDE:
   - NewPing
   - ESP32Servo
   - WiFiManager
   - ArduinoJson
   - PubSubClient

5. Ejecuta el código en el ESP32 con el Arduino IDE

6. Configuración WiFi:
   - Primera vez: Se abrirá un portal WiFi llamado "Configurar-ESP32"
   - Conéctate desde tu celular/PC a esa red
   - Ingresa las credenciales de tu WiFi
   - El ESP32 se conectará automáticamente en futuros reinicios
   - Para resetear WiFi: Mantén presionado el botón en GPIO 0 al encender

7. Verifica en el Monitor Serial (115200 baud):
   - Conexión WiFi exitosa
   - Conexión a HiveMQ exitosa
   - Lecturas de sensores cada 500ms
